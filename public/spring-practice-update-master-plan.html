执行任务分解

任务 1：建立素材清单与去重索引

扫描 [extracted_spring_practice](/Users/wongziming/Documents/New%20project%203/extracted_spring_practice)，按“年级 + 月份段 + 正文/答案”建立唯一键。默认 DOCX 与 PDF 是同源文件，不重复计数。

输入目录：
extracted_spring_practice/春季计算练习/
输出建议：
data/derived/spring_practice_manifest.json
目标输出字段至少包含： grade month_range kind (question_book / answer_book) docx_path pdf_path is_duplicate_group is_missing_primary_docx

任务 2：从正文 DOCX 抽取题目结构

只从正文 DOCX 抽取题面，按“年级 -> 月份段 -> 周 -> 天 -> 题目序号”结构化。PDF 只在 DOCX 解析失败时作人工或程序兜底核对。

读取：
extracted_spring_practice/春季计算练习/*.docx
输出建议：
data/derived/spring_practice_questions_raw.json
每条题目至少保留： source_grade source_month_range source_week source_day source_index raw_text normalized_text detected_pattern estimated_type

任务 3：对齐答案册并建立主答案表

答案册作为主参考源。按同一“年级 + 月份段 + 周 + 天 + 题号”去对齐题目与答案。若答案册与正文题数不一致，单独标注异常。

输入：
extracted_spring_practice/春季计算练习/答案/*.docx
输出建议：
data/derived/spring_practice_answers_aligned.json

任务 4：代码复算并生成异常题清单

对能直接计算的题，统一由代码复算。禁止人工心算直接写答案。把答案册与代码结果不一致的题全部收敛到异常清单。

输出建议：
data/derived/spring_practice_validation_report.json
public/spring-practice-validation-preview.html
可复算题优先包括： + - × ÷ 连加连减 简单连乘连除 部分小数/百分数基础运算 可直接解的一元一次方程 不可稳定复算的题： 分解质因数并求因数个数 数列规律 等差数列求和 公式结构残缺题

任务 5：现有题型映射，不新增题型

把题目映射到当前正式链路稳定支持的 5 类题型：vertical_addition、number_comparison、text_to_number、counting、input。复杂题默认优先落到 choice 的规划维度里，但因为当前正式组件未接入 choice，若本轮禁止改组件，则 choice 题只可出现在规划层，不得正式入库。

参考：
src/data/questions.ts
src/components/QuizScreen.tsx

任务 6：重新设计各年级关卡容量

按前述建议值重排年级关卡数。年级之间不要求统一。幼儿园保留自生成主线，其余年级逐步切到“外部题册主导”。

最终落点：
src/data/questions.ts

幼儿园第 1 关仍然必须保留为演示关卡，包含现有全部稳定题型。不要为了外部题册覆盖率而删除演示关。

在真正覆盖正式题库前，先生成一份新的审稿 HTML，展示融合后的年级、关卡数、来源分布、题型映射、抽样题目。先让人看，再落正式数据。

审稿通过后，才允许回写 src/data/questions.ts。本次只更新数据，不改 UI、不改逻辑、不改组件。

高年级一起做的落地原则

原则 1：一起做，但不强保真所有原题表现形式

高年级内容必须纳入，但如果某题原始形式无法在现有交互中自然承载，则优先做“等价改写”，不是改产品。

原则 2：复杂题先收进候选池

方程、小数分数百分数混合、质因数分解、等差数列等题先收进映射候选池，只有在不改组件的前提下能稳定承载时，才正式入库。

原则 3：本轮“高年级一起做”指数据融合一起做，不是产品能力一起扩

也就是说，高年级内容要进入规划、清洗、校验、预览和正式题库，但不允许借机扩展产品交互。

验收标准

数据层验收

  • 所有正式导入题目都有唯一 ID。
  • 所有正式导入题目都有来源信息，可追溯到年级、月份段、周、天。
  • 所有正式导入题目答案都来自“答案册 + 代码复算”双校验链。
  • DOCX/PDF 没有重复计入题量。
  • 幼儿园第 1 关演示关仍在。

产品边界验收

  • 任何组件文件 diff 都应为 0，除非 owner 单独批准。
  • 任何样式文件 diff 都应为 0。
  • 任何图片资源 diff 都应为 0。
  • 导航流程、地图、结果页、宠物系统 diff 都应为 0。

预览层验收

  • 必须有一份融合审稿 HTML。
  • 必须能看到每个年级最终关卡数。
  • 必须能抽样查看各年级代表题。
  • 必须能看到异常题和未承载题统计。
