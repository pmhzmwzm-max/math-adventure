结论页 · 基于已解压素材与程序化抽样分析

春季计算练习融合设计结论

本页聚焦 4 个核心顾虑：
1. 如果减少原来我们自生成的题目，最少需要保留多少关卡。
2. 答案册如何作为主参考并做二次校验。
3. `docx/pdf` 如何视为同源文件并避免重复。
4. 引入四到六年级后，当前题型交互是否能 cover，以及是否要新增题型。
这版结论默认把高年级一起纳入，不拆阶段。

核心摘要

素材覆盖年级 1-6年级
已确认正文题册 10+ 份
推荐关卡策略 按题量定级
推荐答案来源 答案册主校验
去重原则 按年级+月份段去重
题型结论 建议补 1 个新题型

一句话判断

关卡数

不能再按“每个年级统一 50 关”来做，否则只能吸收很少一部分素材。

答案策略

优先参考答案册是对的，但仍然要由代码复算做二次校验，不能直接照抄。

格式策略

`docx` 作为主解析源，`pdf` 作为版式兜底核对源，不双算题量。

交互策略

低中年级大部分可承接，高年级若追求高保真融合，建议新增 `expression_input`。

2. 答案册如何作为主参考

1
正文题册抽题
从 `docx` 提取题面与题型结构，不从答案册反推题目。
2
答案册配对
用同年级、同月份段、同周同天去对齐答案，答案册作为主答案来源。
3
代码复算
对可计算题自动求值，与答案册对比，发现异常题。
4
异常题复核
对公式丢失、题面不完整、答案不一致的题单独入人工审核池。

主结论

答案册应当作为主参考，但不能直接复制入库，必须经代码二次校验。

可直接复算的题

加减乘除、连加连减、连乘连除、小数/百分数基础运算、部分简易方程。

不宜直接自动复算的题

分解质因数并求因数个数、数列规律、等差数列求和、题面被公式结构打断的复杂题。

当前题型足够覆盖

幼儿园到三年级，大体可以靠现有 `input / vertical_addition / number_comparison / text_to_number / counting / choice` 承接。

高年级想一起做完，建议补一个题型

建议新增 `expression_input`。这是高年级最值当的扩展，只加一个就能显著提升四到六年级承接率。

`expression_input` 最小能力范围

支持 `.`、`/`、`%`、可选字母 `x`，仍然走键盘输入式交互，不必完全重做答题界面。

最终推荐方案

方案结论

高年级要一起做，但不要强行套进原有“统一关卡数 + 统一旧题型”的框架。应按素材题量给各年级独立定关卡数，并新增一个高年级表达式输入题型。

推荐落地顺序

`先做融合规划表 -> 再做抽题与答案配对 -> 再补 expression_input -> 最后正式入库`。
